## Контекст

Из анализа флоу — гэп **J**. T-027 закрывает session-start (что при входе). Обратная сторона — session-end: что должно быть сделано до того, как пользователь ушёл?

Сейчас: `## Лог` таска может отставать, `in_progress` без записи «остановились на шаге N», следующей сессии придётся угадывать. Хочется явный ritual «перед тем как молчать 20+ минут — обнови лог, зафиксируй следующий шаг, отметь блокеры».

## Acceptance criteria

- [ ] В CLAUDE.md добавлено правило session-end: если работа над `in_progress`-задачей приостанавливается (пользователь ушёл на >20 минут / переключается на другую задачу / просит паузу) — обновить `## Лог` таска с записью «остановились на: ... Следующий шаг: ... Блокеры: ...».
- [ ] Триггер: последнее сообщение пользователя ≥ 20 минут назад ИЛИ пользователь явно сказал «на сегодня всё» / «пауза» / «переключаемся».
- [ ] Правило зеркалено в harness template.

## План

1. Обсудить с пользователем: как определять «сессия закончилась» без явного сигнала.
2. Обновить правила.
3. Push.
