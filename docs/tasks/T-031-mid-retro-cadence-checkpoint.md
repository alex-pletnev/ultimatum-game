## Контекст

Из анализа флоу — гэп **G**. Сейчас `mid-retro` — реактивный: срабатывает на 3+ Edit подряд, 2+ ошибок, >15 мин на одну проблему. Проблема: для **тихой длинной работы** (например, пишу большой файл 40 минут, всё компилируется, тесты не падают) — ни один триггер не сработает. Могу спокойно уйти в сторону от изначальной задачи, не заметив.

Нужен дополнительный time-based / cadence-based триггер: например, каждые N последовательных Edit/Write в одном файле или суммарно >30 минут работы без чата с пользователем — пауза.

## Acceptance criteria

- [ ] `.claude/skills/mid-retro.md` дополнен новым триггером `Д (cadence)`: >30 минут работы без обмена репликами с пользователем ИЛИ >7 подряд Edit/Write в `src/**` — обязательный проход, даже если всё компилится.
- [ ] Формулировка cadence-триггера такая, чтобы её мог применить сам агент (нет доступа к таймерам — считаем через размер накопленного диалога / количество tool-calls).
- [ ] То же зеркалено в harness.

## План

1. Обсудить с пользователем формат подсчёта cadence (по tool-calls, по объёму edit'ов, по времени).
2. Обновить `mid-retro.md` в проекте и harness.
3. Push.
